onem2mlib module

This Python3 module implements a library to access and manage resources on a oneM2M CSE.

Functions

def retrieveResourceFromCSE(

parent, resourceID)

Retrieve a resource by its resourceID from the CSE. Any valid parent resource instance from that CSE must be given as the first parameter to pass on various internal attributes. The type of the resource is determined during retrieval.

When successful, this method returns the retrieved resource, or None otherwise.

class AccessControlRule

This class provides the structure for access control rules that shall be used to define privileges in <accessControlPolicy> resources. It contains:

  • accessControlOriginators : The accessControlOriginators is a mandatory parameter in an AccessControlRule. It represents the list of Originators that shall be allowed to use an access control rule. The list of Originators is described as a list of parameters, where the types of the parameter can vary within the list. See also table Table 9.6.2.1-1 of oneM2M's TS-0001.
  • accessControlOperations : The accessControlOperations is a mandatory parameter in an AccessControlRule structure that represents the set of operations that are authorized using this access control rule.

class ContentInstance

This class implements the oneM2M <contentInstance> resource. This type of resource can only be created or deleted, but not updated.

It is usually a sub-resource of the <container> resource.

To publish data in a <contentInstance> resource, one must first encode the data so that it can be transfered as a string value and must then set the encoding type in the contentInfo state variable. The default is text/plain:0. See also the oneM2M specification for further details.

To retrieve the actual content value of the resource (from content) one must first check the contentInfo state variable to determine the type of the content and decode it accordingly.

class Group

This class implements the oneM2M <group> resource.

The <group> resource represents a group of resources of the same or mixed types. The <group> resource can be used to do bulk manipulations on the resources represented by the memberIDs attribute. The <group> resource contains an attribute that represents the members of the group and the <fanOutPoint> virtual resource that enables generic operations to be applied to all the resources represented by those members.

Parent Classes

  • ResourceBase

Instance variables

var consistencyStrategy

Integer. This attribute determines how to deal with the <group> resource if the memberType validation fails. Its possible values (from the onem2mlib.constants sub-module) are

  • Grp_ABANDON_MEMBER : delete the inconsistent member
  • Grp_ABANDON_GROUP : delete the group
  • Grp_SET_MIXED : set the memberType to "mixed"

The default is Grp_ABANDON_MEMBER.

var currentNrOfMembers

Integer. Current number of members in a <group>. It shall not be larger than maxNrOfMembers. R/O.

var fanOutPoint

String. The resourceID of the virtial <fanOutPoint> resource. Whenever a request is sent to the <fanOutPoint> resource, the request is fanned out to each of the members of the <group> resource indicated by the memberIDs attribute of the <group> resource. R/O.

var groupName

String. Human readable name of the <group>.

var maxNrOfMembers

Integer. Maximum number of members in the <group>.

var memberIDs

List String, member resource IDs. Each memberID should refer to a member resource or a (sub-) <group> resource of the <group>.

var memberType

Integer. This is the resource type of the member resources of the group, if all member resources (including the member resources in any sub-groups) are of the same type. Otherwise, it is of type 'mixed'. W/O.

var memberTypeValidated

Boolean, or None. Denotes if the resource types of all members resources of the <group> has been validated by the Hosting CSE. In the case that the memberType attribute of the <group> resource is not 'mixed', then this attribute shall be set.

var resources

List of resource instances. The resources in this <group>.

Create a <subscription> to resource and receive notifications. For this, the notification sub-module must be enabled, ie. onem2mlib.notifications.setupNotifications() must have been called sucessfully. To stop notification from the resource and to remove the subscription, the unsubscribe() method must be called.

This method might throw a onem2mlib.exceptions.NotSupportedError exception in case the target resource type doesn't support subscriptions.

Note

The <subsription> resource created with this method is only valid for the runtime of the calling program. The scubscription will be removed at least when the program terminates, or when onem2mlib.notifications.shutdownNotifications() is called.

class Session

A Session object is used when connecting to a oneM2M CSE. It holds various information about the current session, such as the CSE endpoint, credentials, desired encoding, etc.

Instance variables

String. The URL of the CSE host to connect to. The address includes the protocol, hostname, port number, and any API prefix etc.

var encoding

Integer, either onem2mlib.constants.Encoding_XML or onem2mlib.constants.Encoding_JSON. It specifies the type of encoding for requests between the AE and the CSE.

var originator

String. This specifies the originator for identification in access control policies. It can be a domain, an originatorID, the string "all", or a role-ID.

This attribute indicates a notification content type that shall be contained in notifications. The allowed values are one of the following constants:

  • Sub_AllAttributes (the default)
  • Sub_ModefiedAttributes
  • Sub_ResourceID

var notificationForwardingURI

The attribute is a forwarding attribute that is present only for group related subscriptions. It represents the resource subscriber notificationtarget.

var notificationURI

A list consisting of one or more targets that the Hosting CSE shall send notifications to. A target is either a oneM2M compliant Resource-ID (either structured or unstructured), or as a URL with one of oneM2M supported protocol binding, e.g. http.

var subscriberURI

This attribute is configured with the target of the subscriber. The target is used by the CSE to determine where to send a notification when the subscription is deleted. A target is either a oneM2M compliant Resource-ID (either structured or unstructured), or as a URL with one of oneM2M supported protocol binding, e.g. http.
